# FleetDash – High-Throughput Event-Driven Fleet Telemetry Dashboard

FleetDash is a real-time fleet monitoring platform designed for logistics and transportation companies to efficiently track vehicle movement, monitor fleet activity, and process high volumes of telemetry data.

The project focuses on handling continuous GPS updates from multiple vehicles, storing telemetry efficiently using the MongoDB Bucket Pattern, and providing real-time insights through an interactive dashboard.

FleetDash follows an event-driven architecture, making it scalable and suitable for applications where thousands of telemetry events are generated every minute.

## Problem Statement

Traditional fleet tracking systems often struggle with storing and processing large volumes of telemetry data generated by moving vehicles. As the number of vehicles increases, database operations become slower, affecting real-time monitoring and analytics.

FleetDash addresses this challenge by using an optimized data storage strategy and a modular backend architecture capable of processing high-frequency telemetry efficiently.

## Workflow

1. Vehicle devices continuously generate telemetry data.
2. The backend receives telemetry events.
3. Worker Threads parse and validate incoming data.
4. Telemetry is stored using the MongoDB Bucket Pattern.
5. REST APIs provide processed data to the frontend.
6. The dashboard displays fleet activity and vehicle information.
